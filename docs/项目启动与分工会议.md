# 明舟北渡 · 角色智能体项目启动会

**会议目标**：明确 6 人技术团队分工、技术架构、接口契约与 10 天冲刺里程碑。

---

## 一、项目目标

开发基于剧本《明舟北渡》的角色对话智能体手机端 Web 产品。用户选择角色后，可与 AI 驱动的历史人物进行沉浸式对话，在对话中自然植入大运河文化知识。

**核心交付**：5 个可对话的剧本角色 + 1 个可在手机浏览器访问的 H5 产品。

---

## 二、技术架构（已冻结，10 天内不变更）

```
用户手机浏览器
      ↓ HTTPS
    Nginx（443）
      ├── /api/* → FastAPI 后端 (:8000)
      └── /*     → React 前端静态资源

后端内部
      ├── PostgreSQL（数据持久化）
      ├── Redis（会话缓存，TTL 2h）
      └── DeepSeek API（LLM，备用通义千问）
```

| 层级 | 选型 |
|------|------|
| 前端 | React 18 + Vite + Tailwind CSS + Zustand + React Query |
| 后端 | Python 3.11 + FastAPI + SQLAlchemy + Alembic |
| 数据库 | PostgreSQL 15 + pgvector |
| 缓存 | Redis 7 |
| LLM | DeepSeek-V3（主）/ 通义千问 Max（备） |
| 部署 | Docker Compose + Nginx + 云服务器 |

## 四、接口契约（前后端对接的唯一依据）

**基地址**：`/api/v1`
**响应包装**：`{ code: 0, message: "success", data: {} }`

### 接口清单

| 方法 | 路径 | 说明 | 负责人 |
|------|------|------|--------|
| GET | `/characters` | 角色列表 | 后端主程 |
| GET | `/characters/:id` | 角色详情 | 后端主程 |
| POST | `/sessions` | 创建会话 | 后端主程 |
| POST | `/chat` | **发送消息（核心）** | 后端主程 |
| GET | `/chat/history` | 对话历史 | 后端主程 |

### 核心接口详细定义

**POST /chat**

```json
// Request
{
  "session_id": "sess_xxx",
  "message": "林姑娘，你脸上的痕迹是怎么回事？",
  "client_timestamp": 1716103200
}

// Response
{
  "code": 0,
  "data": {
    "message_id": "msg_xxx",
    "role": "assistant",
    "content": "这痕迹……是前年在百步洪救一个落水孩童时留下的。",
    "character_id": "lin-wenqi",
    "emotion_tag": "hesitant",
    "plot_hook_triggered": null,
    "created_at": "2026-05-19T10:00:05Z"
  }
}
```

**GET /chat/history?session_id=xxx&limit=50&offset=0**

```json
{
  "code": 0,
  "data": {
    "messages": [
      { "role": "user", "content": "..." },
      { "role": "assistant", "content": "...", "emotion_tag": "calm" }
    ],
    "total": 12
  }
}
```

## 八、风险控制

| 风险 | 监控点 | 应对预案 |
|------|--------|---------|
| LLM API 延迟/故障 | Day 3 观察响应时间 | 接入备用模型；兜底文案："（角色望着河水，一时陷入沉思……）" |
| Prompt 效果不达预期 | Day 2 评测 | 立即换模型（DeepSeek→Claude）；增加 Few-shot 到 30 轮 |
| 前端移动端兼容性 | Day 7 真机测试 | Safari 问题多则降级为非流式对话、简化动画 |
| 数据库性能瓶颈 | Day 7 压测 | messages 表加 `session_id` 索引；Redis 缓存历史记录 |
